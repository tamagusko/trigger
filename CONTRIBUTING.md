# Contributing to TRIGGER Open edX

Thank you for your interest in contributing to the TRIGGER Open edX project!

## How to Contribute

### Reporting Issues

If you encounter problems:

1. Check existing issues first
2. Use the issue template
3. Include:
   - Description of the problem
   - Steps to reproduce
   - Expected vs actual behavior
   - Environment details (OS, Tutor version, etc.)
   - Relevant logs or screenshots

### Suggesting Enhancements

For feature requests or improvements:

1. Open an issue with the "enhancement" label
2. Describe the proposed feature
3. Explain the use case and benefits
4. Provide examples if possible

### Submitting Changes

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
   - Follow existing code style
   - Update documentation
   - Test thoroughly
4. **Commit your changes**
   ```bash
   git commit -m "Add: Brief description of changes"
   ```
5. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```
6. **Create a Pull Request**
   - Describe your changes
   - Reference related issues
   - Explain testing performed

## Development Guidelines

### Documentation

- Keep documentation up-to-date
- Use clear, simple language
- Include examples
- Update README.md for major changes

### Scripts

- Follow bash best practices
- Include error handling
- Add comments for complex logic
- Update script README.md
- Test on both local and production environments

### Configuration

- Never commit sensitive data
- Use example files for templates
- Document all configuration options
- Test configurations before committing

## Testing

Before submitting:

1. **Test locally**
   - Verify scripts run without errors
   - Check documentation for accuracy
   - Test on clean environment if possible

2. **Test on production-like environment**
   - AWS or similar cloud provider
   - Test deployment procedures
   - Verify backups and restores

3. **Code review**
   - Self-review changes
   - Check for security issues
   - Verify no secrets committed

## Code of Conduct

- Be respectful and inclusive
- Focus on constructive feedback
- Help others learn and grow
- Follow project guidelines

## Questions?

For questions about contributing:
- Open a discussion issue
- Contact project maintainers
- Check project documentation

Thank you for contributing to TRIGGER!
